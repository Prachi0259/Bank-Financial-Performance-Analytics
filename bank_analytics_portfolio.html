Python Code — Feature Engineering
Anomaly detection using standard deviation method
# Fix date columns
df['TransactionDate'] = pd.to_datetime(df['TransactionDate'])

# Feature: Balance utilisation %
df['BalanceUtilisationPct'] = round(
  (df['TransactionAmount'] / df['AccountBalance']) * 100, 2)

# Feature: Anomaly flag (2 std deviations)
mean_txn = df['TransactionAmount'].mean()
std_txn  = df['TransactionAmount'].std()
df['IsAnomaly'] = df['TransactionAmount'].apply(
  lambda x: 1 if abs(x - mean_txn) > 2 * std_txn else 0)

# Feature: High risk (anomaly + login attempts)
df['IsHighRisk'] = (
  (df['IsAnomaly'] == 1) & 
  (df['LoginAttempts'] > 2)).astype(int)

SQL — 5 Advanced Queries
GROUP BY · WINDOW FUNCTION
Query 1 — Monthly Transaction Volume Trend
SELECT
  MonthName, Month,
  COUNT(*) AS total_transactions,
  ROUND(SUM(TransactionAmount), 2) AS total_volume,
  ROUND(AVG(TransactionAmount), 2) AS avg_transaction,
  ROUND(SUM(TransactionAmount) / SUM(SUM(TransactionAmount)) OVER() * 100, 1) AS pct_of_annual_volume
FROM bank_clean
GROUP BY MonthName, Month
ORDER BY Month;
💡
August and September are peak months at $71K and $72K respectively. May shows +53.3% MoM jump — the sharpest spike in the year.
RANK() WINDOW FUNCTION
Query 2 — Top 10 High-Value Customers by Balance
SELECT
  AccountID, CustomerOccupation,
  ROUND(AVG(AccountBalance), 2) AS avg_balance,
  ROUND(SUM(TransactionAmount), 2) AS total_transacted,
  COUNT(*) AS total_transactions,
  ROUND(AVG(BalanceUtilisationPct), 2) AS avg_utilisation_pct,
  RANK() OVER (ORDER BY AVG(AccountBalance) DESC) AS balance_rank
FROM bank_clean
GROUP BY AccountID, CustomerOccupation
ORDER BY avg_balance DESC
LIMIT 10;
💡
All top 10 high-balance accounts belong to Doctors, avg balance ~$14,700 — nearly 3x the overall average of $5,114. Utilisation is only 10–13%, indicating conservative spending.
PARTITION BY
Query 3 — Transaction Breakdown by Location & Channel
SELECT
  Location, TransactionType, Channel,
  COUNT(*) AS total_transactions,
  ROUND(SUM(TransactionAmount), 2) AS total_volume,
  ROUND(AVG(TransactionAmount), 2) AS avg_amount,
  ROUND(100.0 * COUNT(*) /
    SUM(COUNT(*)) OVER(PARTITION BY Location), 1) AS pct_within_location
FROM bank_clean
GROUP BY Location, TransactionType, Channel
ORDER BY total_volume DESC
LIMIT 15;
💡
ATM dominates all top locations. Oklahoma City Online shows highest avg transaction at $432 — online users spend significantly more per transaction than ATM users.
HAVING · AGGREGATION
Query 4 — Anomalous Accounts by Risk Level
SELECT
  AccountID, CustomerOccupation, Location,
  COUNT(*) AS total_transactions,
  SUM(IsAnomaly) AS anomalous_txns,
  SUM(IsHighRisk) AS high_risk_txns,
  ROUND(AVG(TransactionAmount), 2) AS avg_txn_amount,
  ROUND(AVG(LoginAttempts), 2) AS avg_login_attempts,
  ROUND(SUM(IsAnomaly) * 100.0 / COUNT(*), 1) AS anomaly_rate_pct
FROM bank_clean
GROUP BY AccountID, CustomerOccupation, Location
HAVING anomalous_txns > 0
ORDER BY anomalous_txns DESC
LIMIT 10;
💡
Top 3 accounts have 100% anomaly rate. AC00083 (Student, San Diego) averages $1,531 per transaction — strongest fraud signal. Retired customers appear 4 times in top 10 — most vulnerable segment.
CTE + LAG() WINDOW FUNCTION
Query 5 — Month-over-Month Growth Rate
WITH monthly_volume AS (
  SELECT Month, MonthName,
    ROUND(SUM(TransactionAmount), 2) AS total_volume,
    COUNT(*) AS total_transactions
  FROM bank_clean
  GROUP BY Month, MonthName
),
growth AS (
  SELECT Month, MonthName, total_volume,
    LAG(total_volume) OVER (ORDER BY Month) AS prev_month_volume,
    ROUND((total_volume - LAG(total_volume) OVER (ORDER BY Month)) /
      LAG(total_volume) OVER (ORDER BY Month) * 100, 1) AS mom_growth_pct
  FROM monthly_volume
)
SELECT * FROM growth ORDER BY Month;
💡
Most advanced query — double CTE + LAG() Window Function. May +53.3% is the peak. Aug-Sep show back-to-back +21% growth — the strongest consecutive run of the year.
